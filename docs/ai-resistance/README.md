# salarykorea — AI 내성 재포지셔닝 전략 (AI-Resistance Strategy)

> **작성일:** 2026-04-14
> **배경:** AI(ChatGPT, Claude 등)로 대부분의 정보성 질문이 해결되는 시대에, "계산기 사이트"와 "긴 PDF 콘텐츠"가 어떤 차별점을 가질 수 있는가?
> **결론:** 정보 전달 게임에서는 이미 구조적으로 불리함. **"AI가 구조적으로 못 하는 5가지"에 집중해 도구·신뢰·산출물의 3축으로 재포지셔닝**한다.
> **이 문서는 마스터 전략 문서이며, 구체 실행 항목은 본 파일 하단 "실행 체크리스트" 섹션에 있다.**

---

## 1. 문제 정의 — "왜 계산기?" "왜 PDF?"

AI 시대의 구조적 위협:

| 사용자 행동 | AI 시대 | salarykorea의 위치 |
|---|---|---|
| "연봉 5000 실수령액" | ChatGPT 3초 답변 | 계산기 페이지 클릭 → 입력 → 결과 |
| "연말정산 인적공제 뭐뭐?" | AI 요약 즉답 | 긴 가이드 스크롤 |
| "연봉 협상 어떻게?" | AI 대화형 조언 | 40p PDF 독서 |

단순 **"정보 전달"로는 이미 게임이 기울었다.** 정보는 AI가 싸고 빠르게 전달한다.

그러나 AI는 구조적으로 못 하는 것이 있다. **우리의 재포지셔닝은 그 틈에 파고든다.**

---

## 2. AI가 구조적으로 못 하는 5가지 (= 우리의 기회)

### 2-1. 최신성 보증 (Freshness Guarantee)
- AI는 학습 컷오프가 있음. 2026년 개정 세법을 틀리게 답하는 경우가 흔함.
- 특히 한국 세법: 국민연금 상한액, 건강보험료율, 소득세 누진 구간, 신용카드 소득공제 한도 등은 매년 바뀜.
- **salarykorea는 "2026.1.1 시행 기준" 배지를 전면에 박을 수 있음. AI는 못 박음.**

### 2-2. 할루시네이션 리스크 (Hallucination Risk)
- 세금·연봉은 "틀리면 실제 돈이 움직이는" 영역.
- 사람들은 "내 연봉 협상 자료"·"세무서 제출 자료"에서 AI를 신뢰하지 않음.
- **여기서 "검증된 계산기·검증된 숫자"의 자리가 생김.**

### 2-3. 산출물 (Deliverable)
- AI는 채팅. 계산기는 **PDF/이미지/엑셀로 저장 가능한 산출물**을 준다.
- 배우자와 검토, 협상 자리에서 출력, 기록 보관 — AI 채팅으로는 불편.
- **"결과 저장·공유·인쇄" 기능이 AI 답변과 우리의 결정적 차이.**

### 2-4. 의사결정 템플릿 (Decision Framework)
- AI는 "답"을 준다. 사용자는 "결정 프레임"을 원한다.
- BATNA 워크시트, 연봉 협상 스크립트, 연말정산 체크박스 25개 — **채워넣는 것**이지 읽는 것이 아니다.
- AI에게 "내 상황 맞춤 워크시트 만들어줘" 하면 매번 다른 결과가 나오고, 재현·보관·재사용이 어렵다.

### 2-5. 큐레이션 (Curation)
- AI는 "20가지 방법"으로 답해서 오히려 사용자를 마비시킴.
- 전문 콘텐츠의 힘은 **"이 5개만 하세요"라고 빼주는 것**.
- salarykorea가 검증을 거쳐 압축한 판단은 AI의 포괄적 답변과 다른 가치를 제공.

## 3. 재포지셔닝의 3축

### 축 A — 신뢰 레이어 (Trust Layer)
- **검증 배지**: "2026년 개정 세법 기준 · 최종 업데이트 YYYY.MM.DD" 를 계산기 상단에 노출
- **출처·근거 명시**: 결과 하단에 국세청 고시, 국민연금공단 공지, 건강보험공단 요율 링크
- **감수자 표기(가능 시)**: "세무사 OOO 감수" 또는 "국세청 공식 자료 기반" 등
- **업데이트 이력**: 어떤 세율이 언제 변경되어 반영됐는지 작은 changelog

### 축 B — 산출물 (Deliverable)
- **결과 PDF 저장** 버튼: 계산 결과를 A4 인쇄용 PDF로 다운로드
- **결과 이미지 저장** 버튼: 모바일에서 캡처 친화적인 카드 이미지로 저장
- **결과 링크 공유**: URL 파라미터로 계산값을 포함한 공유 링크 생성 (나중에 다시 열 수 있도록)
- **CSV/엑셀 내보내기** (고급): 총보상 계산 등 복잡한 결과는 엑셀로

### 축 C — 도구 (Tool)
- 기존: "읽는 가이드"
- 변경: **워크시트 + 체크리스트 + 템플릿** 중심
- PDF #2(무료 체크리스트)는 이미 이 방향. PDF #1·#3도 **절반 이상을 "채워넣는 워크시트"로** 재설계 (v2 리라이트 필요)

## 4. 구체 실행 로드맵

### Phase 1 — 공통 컴포넌트 설계 (이번 세션 범위)
1. `common.css` 또는 별도 `trust-layer.css`에 재사용 가능한 컴포넌트 스타일 추가
   - `.trust-badge` — 상단 검증 배지
   - `.sources-footer` — 하단 출처 박스
   - `.result-actions` — PDF/이미지 저장 버튼 그룹
2. `trust-layer.js` — 재사용 가능한 JS 유틸
   - `exportResultAsPDF(elementId)` — html2canvas + jsPDF
   - `exportResultAsImage(elementId)` — html2canvas → PNG 다운로드
   - `shareResultURL(params)` — URL 파라미터 기반 공유 링크 생성
3. 파일럿 페이지 1개에 적용 — 샘플로 **income-tax.html** 또는 **salary-calculator**류 추천 (현태님이 우선순위 지정)

### Phase 2 — 파일럿 배포 & 현태님 컨펌 (다음 세션)
- 파일럿 페이지 1개에 3축 모두 적용 → 실사용 테스트
- A/B 요소 결정: 배지 위치, 출처 노출 범위, 저장 버튼 문구

### Phase 3 — 전 계산기 페이지 롤아웃
- 한국어 메인 계산기 약 14~20개부터 시작 (income-tax, insurance, retirement, holiday-pay 등)
- 부동산연구소 계산기도 동일 적용
- 다국어 페이지(en/ja/zh)는 나중에

### Phase 4 — PDF 콘텐츠 v2 리라이트
- 기존 기획안(`docs/premium-pdf/`)을 "긴 글 40p" → "가이드 20p + 워크시트 20p" 구조로 변경
- PDF #1·#3 재기획: 채워넣는 **워크시트·템플릿** 비중 최소 50%
- PDF #2는 이미 이 방향이므로 그대로 진행

### Phase 5 — SEO·랜딩 카피 재작성
- 메타 description, og 태그에 "2026 개정 세법 기준 · 세무사 감수 · 결과 PDF 저장" 문구 추가
- 홈 랜딩에 "왜 AI가 아니라 salarykorea인가" 한 섹션 추가

## 5. AI를 위협이 아닌 동반자로 (선택적 확장)

공격적 옵션: **"ChatGPT에 이 결과 붙여넣을 프롬프트 10개"** 섹션을 각 계산기 페이지에 삽입.
- 예: "계산된 실수령액을 기준으로 월 저축 플랜을 세워줘" 같은 프리셋 프롬프트 + 복사 버튼
- AI를 배척하는 대신 AI의 **진입점**이 되겠다는 포지션
- 리스크: 사이트 체류시간 감소 가능성. → 파일럿 결과 본 후 도입 판단.

## 7. KPI (파일럿 기준)

- **정량**:

- **정성**:
  - 현태님 체감: 페이지가 "AI와 다른 것"으로 보이는가

## 8. 리스크 & 완화

| 리스크 | 완화책 |
|---|---|
| 세법 업데이트 누락 → 배지가 오히려 신뢰 파괴 | 매년 1월 자동 체크리스트 + 변경 감지 스크립트 |
